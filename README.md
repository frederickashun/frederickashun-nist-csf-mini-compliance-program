# NIST CSF 2.0 Mini Compliance Program

## Project Overview
This project demonstrates a beginner GRC-style cybersecurity assessment using the NIST Cybersecurity Framework (CSF) 2.0. I evaluated a fictional small business environment, identified control gaps, documented risks, and recommended practical remediation steps.

## Objective
The objective of this project was to assess cybersecurity maturity, identify missing or partially implemented controls, and communicate findings in a structured way that reflects real governance, risk, and compliance work.

## Company Scenario
The assessed organization is **NorthBridge Legal Services**, a fictional 25-person professional services company that uses:
- Microsoft 365 for email and file sharing
- Company laptops
- Shared cloud storage
- Office Wi-Fi
- Basic antivirus
- Outsourced IT support

The organization handles:
- Client personal information
- Contracts and legal documents
- Employee HR data
- Billing and invoice records

## Scope

### In Scope
- Microsoft 365 email and cloud access
- Employee laptops
- User access controls
- Shared business data
- Endpoint protection
- Backup practices
- Incident response readiness

### Out of Scope
- Physical building security
- Internal systems of third-party vendors
- Customer-owned devices
- Advanced cloud infrastructure

## Framework Used
This project uses the **NIST Cybersecurity Framework (CSF) 2.0** and reviews controls across:
- Govern
- Identify
- Protect
- Detect
- Respond
- Recover

## Files Included
- **01_Project_Overview.docx.pdf** – project background, scope, and framework
- **02_Control_Matrix.pdf.pdf** – control assessment with status, owners, risk levels, and actions
- **03_Gap_Summary.pdf.pdf** – written summary of key findings and recommendations

## Key Findings
The most significant gaps identified were:
- No formal cybersecurity ownership
- No documented security policies
- Incomplete MFA coverage for all users
- No regular security log review process
- No documented incident response plan
- No backup restoration testing

## Recommendations
Priority recommendations include:
1. Assign cybersecurity ownership
2. Develop baseline security policies
3. Expand MFA to all users
4. Establish regular log review
5. Create an incident response plan
6. Perform scheduled backup restoration testing
